通过这个例子可以看出，如果不使用工厂模式，那么我们对UserService对象和对UserDao对象的创建都是硬编码在程序中的。如果此时将UserServiceImpl类替换成UserServiceImplNew，那么就需要对原有代码进行修改。难道每次替换成一个新的类，就要改一次代码吗？这显然是不合理的，存在耦合，违背“开闭原则”。

耦合：指代码间的强关联关系，一方的改变会影响到另一方

耦合带来的问题：不利于代码维护

因此，为了“解耦合”，我们需要使用一个非常重要的设计模式——工厂模式。

但是这种做法，耦合看似解除了，但其实还是存在的，只是转移到工厂类中了。在工厂类中，我们还是会通过new去创建对象，以硬编码的形式写在程序中。因此，接下来我们需要对工厂类再次解耦。

全限定类名：全限定类名（Fully Qualified Class Name）是指一个 Java 类的完全限定名，包括包名和类名。在 Java 中，类名必须是唯一的，而包可以有相同的名称。因此，使用全限定类名可以确保唯一地标识一个类。例如：com.wyd.dao.UserDaoImpl。
